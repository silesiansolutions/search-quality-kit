# Proposed v0.7 direction

v0.7 should focus on operating the existing checks across more sites, not expanding the SEO rule catalog:

- monorepo and multi-site runner;
- aggregated reports with per-site attribution;
- per-site baselines and isolated failure policy;
- a small public documentation site or landing page;
- better bounded SARIF and GitHub annotations where source locations are honest;
- optional Search Console integration as a separate package/plugin, never a core dependency.

This document is planning only. v0.6 does not implement these features, call Google APIs, execute browsers, score content, or measure Core Web Vitals.
